Hash table implementation with separate chaining.

This class provides a generic hash table implementation that serves as the base for hash-\/based containers. It implements\+:
\begin{DoxyItemize}
\item Key-\/value pair storage
\item Dynamic resizing
\item Basic hash table operations
\end{DoxyItemize}

Performance Characteristics\+:
\begin{DoxyItemize}
\item Insertion\+: Average O(1), Worst O(n)
\item Lookup\+: Average O(1), Worst O(n)
\item Deletion\+: Average O(1), Worst O(n)
\end{DoxyItemize}

The implementation guarantees\+:
\begin{DoxyItemize}
\item Unique keys (no duplicates)
\item Automatic resizing when load factor thresholds are crossed
\item Exception safety (basic guarantee) 
\end{DoxyItemize}

This function performs a deep copy of the entire old\+\_\+buckets array used by the hash table, including the internal linked list in each bucket (used for separate chaining).

Copying strategy\+:
\begin{DoxyItemize}
\item For each bucket index, traverse the linked list of hash nodes
\item Use {\ttfamily \doxylink{classoriginal_1_1hashTable_a1186aae1a84a125368c23008eb37a9cd}{create\+Node()}} to allocate a new node for each entry
\item Preserve the order of elements within each bucket\textquotesingle{}s chain
\end{DoxyItemize}

This method is typically used in\+:
\begin{DoxyItemize}
\item Copy constructors
\item Copy assignment operators
\end{DoxyItemize}

The function ensures that\+:
\begin{DoxyItemize}
\item The original and copied hash tables do not share memory
\item All keys and values are copied correctly
\item \doxylink{classoriginal_1_1hashTable_1_1Iterator}{Iterator} validity is preserved for the new structure
\end{DoxyItemize}

\begin{DoxyNote}{Note}
This function assumes the original old\+\_\+buckets are well-\/formed (no cycles). 

Uses allocator rebound to allocate memory for the new nodes. 
\end{DoxyNote}

Predefined bucket sizes for hash table resizing. 

An array of prime numbers carefully selected for hash table bucket sizes. These primes are used during table resizing to maintain optimal performance characteristics.

Key Properties\+:
\begin{DoxyItemize}
\item All values are prime numbers to reduce hash collisions
\item Each size is approximately double the previous (with some variance)
\item Covers a wide range from small to very large tables
\item Specifically chosen to avoid common modulo patterns
\end{DoxyItemize}

Selection Criteria\+:
\begin{DoxyEnumerate}
\item Primes spaced roughly exponentially (growth factor \texorpdfstring{$\sim$}{\string~}1.8-\/2.\+2)
\item Avoid primes close to powers of 2 to prevent clustering
\item Sufficient gaps between sizes to justify resize operations
\item Includes sizes suitable for both small and large datasets
\end{DoxyEnumerate}

Performance Impact\+:
\begin{DoxyItemize}
\item Larger sizes reduce collisions but increase memory usage
\item Smaller sizes conserve memory but may increase collisions
\item The growth factor balances between resize frequency and memory overhead
\end{DoxyItemize}

The sequence continues until reaching sizes suitable for maximum practical in-\/memory hash tables (over 100 million buckets).

\begin{DoxyNote}{Note}
The actual resize operation only occurs when the load factor exceeds thresholds, not necessarily at every size transition.
\end{DoxyNote}
